# 연구실 홈페이지 빌더

#데이터처리 #정적사이트 #JSON

## 개요 (Overview)

연구실 홈페이지를 정적 웹사이트(HTML/CSS/JS)로 만들되, 구성원·논문/실적·뉴스 등 콘텐츠는 코드에 직접 넣지 않고 JSON 데이터 파일로 분리하여 관리한다. 데이터 파일만 추가·수정하면 홈페이지 화면에 자동으로 반영되는 구조로 설계하여, 본인이 익숙한 "데이터처리" 방식(데이터와 표현 로직 분리)을 웹 개발에 적용해본다.

## 목표 (Goals)

- JSON 데이터를 읽어 화면을 자동으로 구성하는 구조를 직접 구현하고 동작을 확인한다
- 핵심 기능을 먼저 구현하고 검증한 뒤 점진적으로 기능을 추가하는 개발 프로세스를 연습한다
- 개발 지식이 없는 사람도 JSON 파일만으로 콘텐츠를 편집할 수 있는 구조를 만들어본다

## 핵심 기능 정의 (Core Features)

멀티 페이지 정적 웹사이트로 구성하며(소개/연구 분야/구성원/논문·실적/공지사항/연락처를 각각 별도 HTML 페이지로 분리), 콘텐츠는 코드에 직접 넣지 않고 JSON 데이터 파일에서 읽어와 렌더링한다. 데이터 편집은 비개발자(교수님/조교 등)도 수행하므로, 표준 JSON은 주석을 지원하지 않는 점을 감안해 `data/README.md`에 필드별 작성 가이드를 별도로 제공하고, 각 JSON 파일에는 실제로 채워진 예시 항목 1개를 기본으로 남겨둔다.

## 데이터 구조 설계

모든 콘텐츠 데이터는 `data/` 폴더 아래 JSON 파일로 관리하며, 각 파일은 배열(array) 형태로 항목을 담는다. 각 파일에는 실제로 채워진 예시 항목 1개를 기본값으로 남겨두고, 필드 설명·필수 여부·허용값은 `data/README.md`에 별도 정리한다 (Phase 1 항목 9).

### `data/members.json`

```json
[
  {
    "id": "hong-gildong",
    "name": "홍길동",
    "role": "교수",
    "email": "hong@lab.ac.kr",
    "interests": ["데이터처리", "머신러닝"],
    "bio": "한 줄 소개"
  }
]
```

| 필드 | 필수 | 허용값/형식 |
|---|---|---|
| `id` | 필수 | 영문·숫자·하이픈으로 구성된 고유 값 |
| `name` | 필수 | 문자열 |
| `role` | 필수 | `교수` \| `박사과정` \| `석사과정` \| `학부연구생` 중 하나 |
| `email` | 필수 | 문자열 |
| `interests` | 선택 | 문자열 배열 (없으면 `[]`) |
| `bio` | 선택 | 문자열 (없으면 빈 문자열) |

### `data/publications.json`

```json
[
  {
    "id": "pub-2026-001",
    "title": "논문 제목",
    "authors": ["홍길동", "김철수"],
    "venue": "학회/저널명",
    "year": 2026,
    "type": "저널",
    "link": "https://...",
    "summary": "논문 요약"
  }
]
```

| 필드 | 필수 | 허용값/형식 |
|---|---|---|
| `id` | 필수 | 영문·숫자·하이픈으로 구성된 고유 값 |
| `title` | 필수 | 문자열 |
| `authors` | 필수 | 문자열 배열 |
| `venue` | 필수 | 문자열 |
| `year` | 필수 | 4자리 정수 |
| `type` | 필수 | `저널` \| `학회` 중 하나 |
| `link` | 선택 | URL 문자열 (없으면 빈 문자열) |
| `summary` | 선택 | 문자열 (없으면 빈 문자열) |

### `data/news.json`

```json
[
  {
    "id": "news-2026-001",
    "title": "공지 제목",
    "content": "공지 내용",
    "author": "작성자",
    "publishedAt": "2026-07-20T10:00:00+09:00",
    "updatedAt": "2026-07-21T09:00:00+09:00",
    "link": "https://..."
  }
]
```

| 필드 | 필수 | 허용값/형식 |
|---|---|---|
| `id` | 필수 | 영문·숫자·하이픈으로 구성된 고유 값 |
| `title` | 필수 | 문자열 |
| `content` | 필수 | 문자열 |
| `author` | 필수 | 문자열 |
| `publishedAt` | 필수 | ISO 8601 날짜시각 (정렬 기준으로 사용) |
| `updatedAt` | 선택 | ISO 8601 날짜시각. 값이 있으면 화면에 "수정됨" 표시만 하고 정렬에는 사용하지 않음 |
| `link` | 선택 | URL 문자열 (없으면 빈 문자열) |

### 정렬 규칙

- 구성원: `members.json` 배열 순서 그대로 표시
- 논문: `year` 내림차순 (최신 연도 먼저)
- 뉴스: `publishedAt` 내림차순 (최신 공지 먼저, `updatedAt`은 정렬에 영향 없음)

### 참고사항

- 조회수는 정적 사이트 특성상 서버/DB 없이는 값을 영속 저장할 수 없어 자동 집계가 불가능하므로 이번 범위에서 제외한다.

## 범위 제외 사항 (Out of Scope)

이번 프로젝트에서는 다루지 않는다:

- 구성원 사진, 로고 등 이미지 데이터 처리 (텍스트 데이터만 다룸)
- 서버, 데이터베이스, 백엔드 연동
- 로그인/인증이 필요한 관리자 페이지
- 조회수 자동 집계
- 다국어(한/영) 지원
- 다크모드 지원

Phase 2로 미룬 기능(논문 정렬/필터, 구성원 그룹핑, JSON 에러 표시, GitHub Pages 배포)은 완전히 제외된 것이 아니라 Phase 1 검증 후 추가 예정이다.
