# 데이터 파일 작성 가이드

이 폴더의 JSON 파일을 수정하면 홈페이지 화면에 자동으로 반영됩니다.
코드를 건드릴 필요 없이, 아래 형식에 맞춰 항목을 추가/수정/삭제하면 됩니다.

JSON 파일에는 주석을 쓸 수 없기 때문에, 각 파일의 작성 규칙을 이 문서에 정리했습니다.

## members.json (구성원)

| 필드 | 필수 | 설명 |
|---|---|---|
| `id` | 필수 | 영문·숫자·하이픈으로 만든 고유 값 (다른 항목과 겹치면 안 됨) |
| `name` | 필수 | 이름 |
| `role` | 필수 | `교수`, `박사과정`, `석사과정`, `학부연구생` 중 하나만 사용 |
| `email` | 필수 | 이메일 주소 |
| `interests` | 선택 | 관심 분야 목록. 없으면 `[]` |
| `bio` | 선택 | 한 줄 소개. 없으면 `""` |

## publications.json (논문/실적)

| 필드 | 필수 | 설명 |
|---|---|---|
| `id` | 필수 | 영문·숫자·하이픈으로 만든 고유 값 |
| `title` | 필수 | 논문 제목 |
| `authors` | 필수 | 저자 목록 |
| `venue` | 필수 | 학회 또는 저널명 |
| `year` | 필수 | 발표 연도 (4자리 숫자) |
| `type` | 필수 | `저널`, `학회` 중 하나만 사용 |
| `link` | 선택 | 논문 링크. 없으면 `""` |
| `summary` | 선택 | 요약. 없으면 `""` |

## news.json (공지사항)

| 필드 | 필수 | 설명 |
|---|---|---|
| `id` | 필수 | 영문·숫자·하이픈으로 만든 고유 값 |
| `title` | 필수 | 제목 |
| `content` | 필수 | 내용 |
| `author` | 필수 | 작성자 |
| `publishedAt` | 필수 | 게시 일시 (`YYYY-MM-DDTHH:mm:ss+09:00` 형식). 목록 정렬 기준으로 쓰임 |
| `updatedAt` | 선택 | 수정 일시. 값이 있으면 "수정됨" 표시만 되고 정렬에는 영향 없음. 없으면 `""` |
| `link` | 선택 | 관련 링크. 없으면 `""` |

## 공통 주의사항

- 각 파일은 전체가 하나의 배열(`[ ]`)이어야 합니다. 항목을 추가할 때는 배열 안에 `{ }` 블록을 콤마로 구분해 추가하세요.
- 마지막 항목 뒤에는 콤마를 붙이지 않습니다.
- 값을 수정한 뒤에는 JSON 형식이 깨지지 않았는지(따옴표, 콤마, 괄호) 확인하세요.
